2018-2 HCI Assignment 4.2

PacaLog
2018-11-26
Interface

Refine your project proposal.

유저 분석/연구

타겟 사용자 그룹

타겟으로 한 사용자는 다음과 같다.

  • 사진을 보기 좋아하는 사용자
  • 커플 사진을 많이 찍는 사용자
  • 앨범을 손수 분리하지는 않는 사용자

과거의 타겟과 다르게 세 번째 타겟 사용자를 명시한 이유는 주변 조사를 통해 굳이 손수 앨범 추가 기능을 사용하지 않는 사용자가 많다는 사실과 동시에 그들의 사진들이 일련의 기준(장소, 인물 등)에 따라 자동으로 분류되기를 원한다는 사실을 알게 되었기 때문이다. 이번 프로젝트는 AI가 적용되는 사진 정렬 앱이기 때문에 서로 아귀가 맞겠다는 생각을 했다.

사용자 요구사항

인터뷰를 통해 얻은 사용자 요구사항은 다음과 같다.

주요 요구사항
  • 눈에 잘 들어오게끔 가시성이 확보
  • 사진을 잘 볼 수 있게 하는 직관적인 UI
  • 시간 별로 사진을 잘 분류
  • 인공지능을 사용할 경우 사진의 맥락을 파악해 분류

기능적/UI 요구사항

1) 기능적

  • 비율에 맞게 사진 보여주기.
  • 정렬 기준에 맞게 사진 정렬.
  • 정렬 기준을 사용자가 선택할 수 있음.
  • 검색을 통해 원하는 사진을 찾을 수 있음.

2) 비기능적

  • 사진에 대한 날짜, 장소 등 메타데이터의 폰트는 12로 한다.
  • 전체 화면으로 사진을 표시할 경우 비율에 맞지 않는 바깥 배경은 검은색으로 함.
  • 검색은 3초 이내로 이루어지도록 함.

3) 기타 고려사항

실행 환경

iOS 기기에서 찍은 사진들을 사용자가 보고 싶을 때 사용하도록 한다.

기기와 플랫폼

  • iOS 기기
  • iOS 11.0+

기준이나 가이드라인

  • Apple의 Human Interface Guidelines를 고려한다.

사용 시나리오 및 Task model

1) 특정 날짜의 사진 보기

2) 특정 장소의 사진 보기

3) 특정 인물의 사진 보기
