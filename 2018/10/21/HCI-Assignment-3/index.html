사진 정렬 프로그램: 요구 사항과 사용자 분석

A. 요구 사항

1. 응용 프로그램의 전체적인 목적

프로그램의 전체적인 목적은 사용자의 사진을 나름의 기준에 따라 정렬하여 보기 쉽게 하는 것이다. 사용자가 사진을 찍으면 이를 날짜별, 장소별, 인물별 등으로 정리하여 쉽게 사진을 찾아볼 수 있도록 한다.

2. 기능적 요구사항

  • 사진을 눌렀을 때 전체 화면으로 표시
  • 핀치 제스처를 통해 사진을 확대
  • 정렬 기준 조정하기
  • 3D 터치를 통한 사진 미리 보기(peek and pop)

3. 주요 UI 기능

1) 기능적
  • 비율에 맞게 사진을 보여주기
  • 정렬 기준에 맞게 사진을 정렬하기
2) 비기능적
  • 사진에 대한 날짜, 장소 등 메타데이터의 폰트는 12로 한다.
  • 전체 화면으로 사진을 표시할 경우 비율에 맞지 않는 바깥 배경은 검은색으로 한다.

4. 기타 고려사항

1) 사용 환경
  • iOS 기기에서 찍은 사진들을 사용자가 보고 싶을 때 사용
2) 기기와 플랫폼
  • iOS 기기
  • iOS 10+
3) 타겟 사용자 그룹
  • 사진을 보기 좋아하는 사용자
  • 커플 사진을 많이 찍는 사용자
4) 기준이나 가이드라인
  • Apple의 Human Interface Guidelines를 고려한다.

B. 사용자 분석

녹음한 파일은 블랙보드에 올립니다.

1. 유저 인터뷰

인터뷰 링크(사운드 클라우드)

2. 유저의 주요 요구사항

  • 눈에 잘 들어오게끔 가시성이 확보
  • 사진을 잘 볼 수 있게 직관적인 UI
  • 시간 별로 잘 분류
  • 인공지능을 사용할 경우 사진의 맥락을 파악해 분류

3. 인지적 / 인간 공학적 요구사항

  • 사진이 눈에 잘 들어오도록 가시성이 좋아야 함
  • 다양한 기준이 있겠지만 시간 순으로 분류하는 것을 중시
  • 맥락을 파악하는 것을 기대(인공지능이 들어갈 경우)

4. Usage 시나리오

1) 특정 날짜의 사진 검색
2) 특정 장소의 사진 보기

5. Interation Model

1) Task Model #1: 사진 앨범 추가
  1. 앱을 실행한다.
  2. 앨범 탭을 누른다.
  3. ‘플러스’ 버튼을 누른다.
  4. 텍스트 입력 창이 뜨면 앨범의 이름을 설정한다.
  5. 앱을 종료한다.
2) Task Model #2: 인물 검색
  1. 앱을 실행한다.
  2. 검색 탭을 누른다.
  3. 검색어 입력 필드에 특정 인물의 등록된 이름을 입력한다.
  4. 검색 버튼을 눌러 검색을 실행한다.
  5. 앱을 종료한다.
